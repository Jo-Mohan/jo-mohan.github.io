An Exploration on the Math of Procedural Generation

October 2025 · By Jo Mohan

Introduction

Procedural generation is a mathematical technique used in game development to create vast worlds through a set of rules rather than manual modeling. Instead of outlining every detail for the terrain, devs can uses algorithms that simulate natural variation.

Core Generation Concepts

Perlin noise is a gradient-based noise function that produces continuous, flowing patterns rather than abrupt randomness. It assigns random gradient vectors to lattice points on a grid and interpolates between them, yielding smooth transitions. In mathematical terms, we can view the noise as a function \(\mathcal{N}(x, y)\) built from multiple octaves with each adding finer detail at a higher frequency but lower amplitude:

\[ \mathcal{N}(x, y) = \sum_{i=1}^{k} A_i \, \mathrm{noise}(f_i x, f_i y) \] where \(A_i\) controls the amplitude and \(f_i\) the frequency of the \(i^{\text{th}}\) octave. This hierarchical combination creates the familiar “coherent noise” textures used in terrain, clouds, and water.

Fractals are functions that exhibit self-similarity across scales. A common example is the fractal Brownian motion (fBm), which recursively sums scaled copies of noise functions:

\[ \mathcal{F}(x, y) = \sum_{i=1}^{k} \frac{1}{2^i} \, \mathcal{N}(2^i x, 2^i y) \] This generates structures with repeating detail at every zoom level which is useful for something self similar such as a coastline.

We can also blend different terrain characteristics using a fade parameter \(\alpha(x, y)\) to smoothly transition between noise and fractal regions:

\[ H(x, y) = \alpha(x, y)\,\mathcal{N}(x, y) + \big(1 - \alpha(x, y)\big)\,\mathcal{F}(x, y) \] Here, \(H(x, y)\) represents the resulting height map and allows for a linear combination of noise and fractal.
